# ToolkitPlugin for Obsidian

ToolkitPlugin 是一款为 Obsidian 打造的综合功能增强插件，旨在通过一系列实用的微调和自动化功能来提升您的笔记工作流。

目前它主要解决了从 **Wolai** 等块式笔记系统导入数据后的结构痛点，并提供了自由的视觉控制。

## 🚀 主要功能

### 1. 文件夹快速新建 (Quick Create Button)
在文件夹右侧添加一个 `+` 图标（悬停时显示），点击即可在文件夹内快速新建笔记。
- **自动展开**：如果文件夹处于收起状态，点击 `+` 会自动将其展开。
- **焦点定位**：新建笔记后，文件树会自动定位并高亮显示该笔记，方便立即开始编辑。
- **智能拦截**：针对已展开的文件夹，点击 `+` 绝不会触发误收起。

### 2. 文件夹自动打开 (Auto-open Wolai-style Folders)
针对特定的文件夹结构（文件夹名与内部唯一的 `.md` 文件名一致），实现“点击/双击”文件夹直接打开正文的功能。
- **自动识别**：智能匹配文件名与目录名。
- **干扰排除**：自动忽略 `image` 文件夹，确保在 Wolai 风格结构下依然能一键触达内容。

### 3. 文件拖拽锁定 (Drag-and-Drop Lock)
防止在文件列表中浏览时误触导致的文件/文件夹移动。
- **即时锁定**：开启后，文件树内的所有拖拽操作将被拦截并提示。
- **便捷切换**：通过侧边栏的“锁”图标或命令面板快速加锁/解锁。

### 4. 视觉清理 (Visual Cleaning)
一键隐藏/显示文件列表中的特定资源目录。
- **全局隐藏**：默认支持匹配所有层级的 `image` 文件夹（不分大小写）。
- **快捷切换**：通过 Ribbon 图标、命令面板或快捷键即时切换。

### 5. Mermaid 流程图适配与全屏查看 (Mermaid Scaling & Fullscreen)
解决 Mermaid 流程图显示受限的问题，提供多维度的查看方案。
- **自适应宽度**：默认自动缩放至笔记宽度，美观统一。
- **互动缩放**：控制条支持实时 `+` / `-` 缩放，鼠标离开自动恢复。
- **一键全屏**：新增 `Maximize` 按钮，点击可在沉浸式弹窗中查看图表原始全尺寸。
- **手势支持**：全屏模式下支持 **Cmd/Ctrl + 滚轮**（或触摸板捏合）进行缩放，支持 **左键抓取** 进行平移。
- **灵敏度可调**：可在设置中调节 **Mermaid Zoom Sensitivity**（默认 1.0），适配不同触控板手感。
- **独立开关**：可在设置中切换适配功能的开启状态。

### 6. 代码块交互增强 (Code Block Enhancements)
针对 Markdown 代码块的输入与粘贴体验进行了深度优化。
- **自动语言识别**：当向空白代码块（ ` ``` ` ）粘贴代码时，插件会自动识别并补全语言标识符（支持 JS, TS, Python, Java, Kotlin, Bash, HTML, CSS, SQL, Dart）。
- **智能焦点跳转**：当检测到插入空白代码块结构（ ` ```\n\n``` ` ）时，光标会自动跳转到代码块内部，无需手动切换行。
- **高精度识别**：针对 Python 与 Java 等相似语法进行了特征识别优化，大幅减少误判。

### 7. 多语言支持 (Multi-language Support)
插件界面（设置、命令、提示等）会自动适配 Obsidian 的界面语言。
- **支持语言**：简体中文 (zh)、繁体中文 (zh-tw)、英语 (en)。
- **自动回退**：若当前语言不被支持，将自动显示英文界面。

### 8. 高度可配置
所有功能都可以在插件设置页面进行独立开关，满足不同场景下的需求。

## 🛠️ 安装说明

1. 在您的 Obsidian 库中创建目录：`.obsidian/plugins/obsidian-toolkit-plugin/`。
2. 将以下文件放入该目录：
   - `main.js`
   - `manifest.json`
   - `styles.css`
3. 在 Obsidian 设置中启用 **ToolkitPlugin**。

## ⚒️ 开发与构建

如果您想自行修改并构建插件，可以参考以下命令：

1. **安装依赖**：
   ```bash
   npm install
   ```

2. **开发模式 (自动编译)**：
   ```bash
   npm run dev
   ```

3. **生产构建**：
   ```bash
   npm run build / node esbuild.config.mjs production
   ```
   构建完成后，新的 `main.js` 将出现在项目根目录下。
